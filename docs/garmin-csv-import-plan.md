# План: Поддержка второго формата Garmin CSV (Laps export)

## Context

Текущий `importGarminCSV()` в `app.js` работает только с одним форматом экспорта Garmin
(сплиты по интервалам, где `Summary` находится в колонке `r[1]`).
Нужно научиться загружать **экспорт по кругам (Laps)** — формат, в котором каждая строка
это отдельный круг (1 км), а итог — последняя строка `Summary` в колонке `r[0]`.

Пример нового формата — `D:\Downloads\activity_22885509734.csv` (10×1 км + 2 финальных, итого 10.45 км / 1:01:05).

---

## Что ломается в текущем парсере

В новом формате три отличия:

### 1. Заголовки с переносами строк внутри ячеек CSV
В файле есть:
```
"Distance
km","Avg Pace
min/km",...
```
То есть название колонки и единица измерения разделены `\n` **внутри одной CSV-ячейки**.
Текущий `parseCSV()` делает `text.split(/\r?\n/)` ДО разбора кавычек, поэтому первая
строка заголовков разрезается на 5–6 «обрезков», и map `header → index` строится по мусору.

### 2. «Summary» — в колонке 0, а не в колонке 1
Текущий поиск `rows.find(r => r[1] === 'Summary')` не находит строку, потому что в Laps-формате
структура такая:
```
"Summary","1:01:05","1:01:05","10.45","5:51",...
   ↑ r[0]
```

### 3. Имена колонок с хвостом единиц
`Distance` → `Distance\nkm`, `Avg Pace` → `Avg Pace\nmin/km`, и т.д.
Прямой lookup `idx['Distance']` не сработает.

## Изменения (только фронтенд — `app.js`)

### 1. Переписать `parseCSV()` — посимвольный парсер с поддержкой многострочных ячеек

Заменить текущую реализацию на проход по всему тексту одним циклом,
отслеживая `inQuotes` глобально (через переносы строк):

```javascript
function parseCSV(text) {
  const rows = []; let row = []; let cur = ''; let inQ = false;
  for (let i = 0; i < text.length; i++) {
    const ch = text[i];
    if (ch === '"') {
      if (inQ && text[i+1] === '"') { cur += '"'; i++; }   // экранированная кавычка
      else inQ = !inQ;
    } else if (ch === ',' && !inQ) {
      row.push(cur); cur = '';
    } else if ((ch === '\n' || ch === '\r') && !inQ) {
      if (ch === '\r' && text[i+1] === '\n') i++;
      row.push(cur); cur = '';
      if (row.length > 1 || row[0] !== '') rows.push(row);
      row = [];
    } else {
      cur += ch;
    }
  }
  if (cur || row.length) { row.push(cur); rows.push(row); }
  return rows;
}
```

Эта версия:
- сохраняет `\n` внутри ячейки (`"Distance\nkm"` остаётся одной ячейкой со значением `Distance\nkm`)
- обрабатывает экранированную кавычку `""` → `"`
- корректно работает на **обоих** форматах Garmin (старый продолжит работать)

### 2. Нормализовать имена заголовков

После парсинга шапки построить нормализованный индекс — отрезать всё после `\n` и привести к нижнему регистру:

```javascript
function normalizeHeader(h) {
  return h.split(/\r?\n/)[0].trim().toLowerCase();
}
const idx = {};
headers.forEach((h, i) => { idx[normalizeHeader(h)] = i; });

const get = (name) => {
  const i = idx[name.toLowerCase()];
  return i !== undefined ? summary[i] : null;
};
const dist  = get('Distance');
const time  = get('Cumulative Time') || get('Time');
const pace  = get('Avg Pace');
const hr    = get('Avg HR');
const maxHr = get('Max HR');
const asc   = get('Total Ascent');
const cal   = get('Calories');
const cad   = get('Avg Run Cadence');
```

Фоллбэк `Cumulative Time` → `Time` нужен потому, что в Laps-формате значения в этих
колонках одинаковы (общее время забега), но возможны экспорты только с одной из них.

### 3. Искать Summary в колонке 0 ИЛИ 1

```javascript
const summary = rows.find(r => r[0] === 'Summary' || r[1] === 'Summary');
```

## Верификация

1. **Регрессия (старый формат — сплиты):** загрузить файл, в котором Summary в `r[1]` и заголовки в одну строку → форма заполняется как раньше.
2. **Новый формат (Laps):** загрузить `activity_22885509734.csv` →
   - дистанция: `10.45`
   - время: `1:01:05`
   - темп: `5:51`
   - пульс средний: `142`
   - заметки: `Garmin: пульс макс 155, набор 48м, каденс 174, калории 794`
3. **Граничные случаи:** пустой файл, файл без Summary → понятная ошибка в `garmin-msg`.

## Что НЕ делаем сейчас

- Не парсим отдельные круги (Laps) как самостоятельные интервалы — берём только Summary.
- Не добавляем UI выбора формата — определяем автоматически.
- Не меняем бэкенд.
